# Workflow Overview

## What is this Repo?
This is the central archive for all GIDS long-form technical scripts.
It stores, organises, and tracks every piece of content produced by associates
from first draft through to Instagram publication.

## How it's Organised
- `/guides` — How to use this repo (start here if you're new)
- `/post-topics` — All scripts organised by topic
- `ig-adapted.md` — A running list of all published Instagram post links

## What are Issues?
Every script in the archive has its own Issue. This is how we track
where a script is in the process. Each issue includes the script details,
a link to the Google Drive doc, and a status label.

## What are Labels?
Labels are applied to Issues to show the current status of a script.
As a script progresses, the label is updated accordingly.

## What is the Project Board?
The Project board gives a visual Kanban view of all scripts in progress.
Each script card moves across the following columns automatically based on its label:

| Draft | Finalised | Queued for IG | Posted to IG |

## Pipeline
Draft → Finalised → Queued for IG → Posted to IG

## Steps
1. Script is written and uploaded to `/post-topics/` relevant folder
2. An Issue is opened for the script using the Script template
3. The `script-draft` label is applied
4. Once reviewed and approved, label is updated to `script-finalised`
5. When selected for IG, label is updated to `queued-for-ig`
6. Once posted, label is updated to `posted-to-ig` and link is added to `ig-adapted.md`
